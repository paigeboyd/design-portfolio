Thirsty?

As part of a year 2 Integrated Multimedia Development module, we were given a group project to develop a web app which solved a problem. From the outset my group wanted to create something which solved a 'problem' many people faced in their day-to-day lives. Through brainstorming we reached the question of what can be fixed in the mornings when people are going to work or university?

We then reached the decision to create an app called Thirsty? Which is basically a 'fast-lane' in the coffee shop during rush hour. We targeted the app towards professionals and students, who didn't have time to stand in long queues before class or work.


My Role

My role in the group was primarily to be the User Interface Designer. I designed the website with ease of use in mind, enabling users to order their coffee on the go, without needing to go through a complicated process. Obviously speed is an important element to this, so we tried to minimise the process in to as few steps as possible, with the idea that once the user had used the app once, the process would be streamlined, as the app would remember their favourite choice and would store their details, eliminating the longest step.


The Process

I kept the interface clean and simple and developed a template based on the design I created using Adobe Illustrator. I used HTML, CSS and JavaScript to bulk out the template, but felt that more interaction was needed to enhance the overall user experience. To do this I worked with a few jQuery plugins as these handled interactions such as a smooth scroll, hover states and animations on the content once the user scrolled to it.



Final Outcome

As this was a university project, the brief required us to work to a strict time frame, and in an agile environment. This was a new experience for me, and was a great insight into how projects are handled in the real world. With this being such a big task it was interesting to see how my group worked to strict time frames and it taught me a lot about how to prioritise tasks to get the work done on time. I was pleased that I got to undertake the role of UI designer, as this is where I feel my strengths lie. This project is no longer live however you can see the code on GitHub and screenshots below.
